### 視覺與互動判斷

- 橫式 TD：1920px 截圖的中央五路與左右進攻方向保持乾淨；教室場景的高細節集中於邊緣，敵我剪影、陽光與格線可辨識，符合中央玩法區不雜亂的要求。
- 直式 TD：方向門在進入戰鬥時完整遮罩，說明「請橫放手機」並提供旋轉行動；沒有錯把橫式 TD 壓縮成直式戰場。
- 小遊戲：直式黑板構圖與橫式操場構圖為不同點陣背景，中央準星/目標保有足夠對比，未見旋轉共用主圖的跡象。
- VFX 與敵我：預覽截圖的白色範圍線、半透明金色面與敵方圖像能區分；Phase 3 截圖仍可看到格線、Boss、投射物與危險物。但 Boss 名稱/階段標題從右側溢出，破壞關鍵戰況文字的可讀性。
- 一致性：字型、深綠 HUD、木質卡片、圓角按鈕與彩色物件風格一致；間距在寬畫面穩定，窄橫式頂部資訊密度過高。

## Round 2 - 互動資料流追蹤

### 文具大招：按鈕 -> arm -> preview -> confirm/cancel -> runtime

1. 印章按鈕在印章大於 0 時呼叫 `onStampUltimate(toolId)`，再進入 `engine.releaseStampUltimate()` 與 `armStampUltimate()`。
2. `armStampUltimate()` 建立 `pendingUltimate`，預設中央格，並顯示「選擇目標格」浮字。
3. 指標移動更新 `pendingUltimate.previewCell`；`drawUltimateTargetPreview()` 依大招形狀畫出格線與半透明預覽。
4. 格內點擊進入 `confirmStampUltimate()`：先取得合法目標，無目標時不扣印章；有目標才扣一枚印章、套用 AoE/持續效果，並生成 `ultimateFx` 影像階段。
5. 格外點擊與重按同一大招會走 `cancelStampUltimate()`，不扣印章。此路徑確實存在，但沒有畫面上的取消控件或操作提示，造成 H3/H5/H9 的可發現性缺口。

### Boss：telegraph -> action -> vulnerability

1. `transitionBossPhase()` 依 HP phase table 觸發一次，顯示階段預警、生成 telegraph VFX，並推入延遲 `bossAction`。
2. 延遲結束後 `executeBossPhaseAction()` 呼叫遠程彈幕、召喚、危險區或衝刺；遠程攻擊使用 `spawnEnemyProjectile()`，保有發射、飛行與命中生命週期。
3. 動作完成後 `startBossVulnerability()` 設定破綻計時；`damageEnemy()` 在計時內套用 1.35 倍傷害。班主任大招命中 Boss 亦會建立對應破綻。
4. 資料流完整且能對應票面「預警 -> 行動 -> 反擊窗口」；但畫面未保證在窄橫式 viewport 讓 Boss 標題與階段資訊完整可讀。

## Findings

| 嚴重度 / 優先級 | 發現 | 證據與影響 | 改善建議 |
|---|---|---|---|
| Blocking / P0 | 無 | 未發現會讓現有滑鼠/觸控 TD 流程無法開始或讓印章錯誤消耗的證據。 | - |
| Severe / P1 | Canvas 核心戰場無鍵盤等效操作，也沒有關鍵戰況的可讀替代輸出。 | 違反 WCAG 2.1.1；鍵盤/輔助科技使用者無法選格、取消大招或理解預警、破綻與命中。 | 為 Canvas 提供可聚焦的操作層與鍵盤選格/確認/取消；以 aria-live 或等效 DOM 狀態輸出波次、瞄準、預警、破綻與命中。 |
| Severe / P1 | 大招「取消」存在但不可發現。 | 預覽畫面沒有取消按鈕、Esc 提示或「點擊戰場外取消」提示；使用者易把預覽誤認為卡住。 | 在 preview 顯示固定的「取消」按鈕與操作提示（觸控與滑鼠）；支援 Esc 作為鍵盤取消。 |
| Moderate / P1 | 844px 橫式 Boss 場景中 Boss 名稱/階段標題由右側溢出，與頂部 HUD/提示條競爭。 | `xueba_boss_phase3_game.png` 可見紫色 Boss 標題裁切；這正是需要被讀到的 telegraph 資訊。 | 將 Boss 名稱與階段移至可用安全區內，採最大寬度、縮放/換行，並在窄橫式優先保留 Boss 名稱、預警與格線。 |
| Moderate / P2 | 大招點擊到格內但沒有任何敵人時不扣資源，卻沒有可見的恢復提示。 | runtime 回傳 `false` 並保留瞄準狀態；玩家不知道該換格、取消或是否成功。 | 顯示「範圍內沒有目標，印章未消耗」並保留高亮預覽。 |
| Minor / P2 | 窄橫式頂部資訊密度偏高。 | Boss 截圖的波次、資源、教學條、注意條與 Boss 標題同時占用上方區域。 | 僅在首次/狀態變更時顯示教學條，或在 Boss 戰將低優先提示折疊為單行。 |
